Predictive analytics

Campaign Response Modelling (Classification)

I trained and evaluated response models to prioritise customers for outreach, improving targeting efficiency under budget constraints. The focus is decision support: choosing a threshold that balances reach vs performance.

Python Logistic Regression Random Forest ROC-AUC Thresholding
Open full report (HTML) View code (GitHub) Back to home

Notebook: 01_predictive_response.ipynb · Supporting library: VBDA InsightLab

Problem

Business context

Marketing budgets are limited and response is typically imbalanced (few responders). The goal is to rank customers by likelihood to respond, then choose a practical threshold that maximises impact while controlling spend.

Data + method

How it was built

  • Target: campaign response (classification).
  • Models: Logistic Regression (interpretable baseline) + Random Forest (non-linear benchmark).
  • Imbalance handling: class weighting and evaluation beyond accuracy.
  • Evaluation: ROC-AUC, precision/recall, confusion matrix and threshold comparison.
  • Output: ranked customer scores for prioritised targeting.

Insights

3 key takeaways

  1. Ranking beats guessing: model scores enable prioritisation even when overall response is low.
  2. Threshold matters: small threshold changes can significantly affect audience size and efficiency.
  3. Interpretability supports action: baseline drivers help explain why targeting changes.

Recommendations

What I would do next

  • Choose targeting thresholds based on CRM goals (efficiency vs scale), not only on model metrics.
  • Run A/B tests on score-based targeting vs current selection rules to quantify uplift.
  • Monitor drift and refresh the model on a regular cadence as customer behaviour changes.
