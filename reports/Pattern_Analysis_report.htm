Pattern Analysis Customer Value, Channels & Lifecycle
Purpose & scope
This report describes observable customer behaviour patterns to inform CRM planning: customer value, channel preference, and lifecycle timing (join month, tenure, recency). It is descriptive only (no segmentation/clustering, no predictive modelling). Timing analysis is limited to join month, tenure, and recency because transaction timestamps are not available.

How to read the time variables in this analysis

How to read the time variables in this analysis

In this report we use several time-related concepts that look similar but answer different CRM questions.

  • Cohort (Join Month): when the customer joined (join-month wave). Used to compare cohort size and average value at the snapshot date.
  • Tenure: how long the customer has been with us (relationship length). Used to compare behaviour across lifecycle stages (new vs established).
  • Recency: days since last purchase. Used as an operational trigger to prioritise retention and win-back actions.
  • Age: approximate customer age. Used only for light profiling; behaviour (value, channel, activity) remains the primary CRM lever.

Important limitation: Without transaction timestamps, we cannot analyse day-of-week purchasing, time-of-day, or true purchasing seasonality/cadence.
